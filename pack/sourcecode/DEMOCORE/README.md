## 🆕 Architecture Updates (Prof. Hans)

The simulation now implements the **complete Prof. Hans architecture**:

### 1. **QUBO Optimizer** (Bottleneck 1 + NISQ Bridge)
- **MIMO Tensor Decomposition** (Slide 14): Detects smurfing/structuring patterns
  - Analyzes transaction time-series matrix T[wallet, time_bucket]
  - Flags wallets where total ≥ $10k but individual txs < 90% threshold
  - Applies risk boost (up to +0.45) to detected smurfing nodes
- **Top-20 Subgraph Selection** (Slide 8): NISQ-ready subgraph extraction
  - Selects top-K high-risk nodes based on: risk + centrality
  - Ensures target wallet is always included
  - Optimizes for quantum hardware constraints (|V| ≤ 20)
- **F-β Cost Function** (β=0.5): Precision weighted 4x higher than Recall

### 2. **6-Component Compliance Scoring** (Slide 13)
```
Rₐ = ωr·r̃ₐ + ωq·ζₐQ + ωE·Eₐ + ωS·Sₐ + ωC·Cₐ + ωO·Oₐ
```
Where:
- **r̃ₐ** (ωr=0.30): Classical risk score (node features, degree centrality)
- **ζₐQ** (ωq=0.25): Quantum evidence (QUBO optimization result)
- **Eₐ** (ωE=0.20): External sanctions signal (OFAC/EU/UN match)
- **Sₐ** (ωS=0.15): Scam/structuring signal (CryptoScamDB + smurfing)
- **Cₐ** (ωC=0.07): Counterparty exposure (mixer usage)
- **Oₐ** (ωO=0.03): Obfuscation (cross-chain bridge)

All weights sum to 1.0 (calibrated from historical labelled cases).

### 3. **Parametric Thresholds** (Bottleneck 2, Slide 15)
- **τH = 0.75**: HIGH risk → **FREEZE** (immediate action)
- **τM = 0.45**: MEDIUM risk → **MONITOR** (enhanced surveillance)
- **< τM**: LOW risk → **CLEAR** (normal processing)

Sanctions hit **always triggers FREEZE** regardless of score.

### 4. **FATF Travel Rule Compliance** (Recommendation 16)
- Flags transactions > $1,000 USD without VASP originator/beneficiary data
- Meets SEA regulatory requirements (MAS, OJK)
- Included in SAR report for compliance audit trail

## Quick Start

### Open the Prototype (zero setup)
```
Open DEMOCORE/04_prototype.html in Chrome / Firefox / Edge
```
No server, no npm, no API keys. Just open the file.

### Run the AI/ML Simulation (Python only)
```bash
# Analyze a specific wallet
py 03_qubo_sim.py 0xd90e2f925DA726b50C4Ed8D0Fb90Ad053324F31b

# Get JSON output
py 03_qubo_sim.py 0xd90e2f925DA726b50C4Ed8D0Fb90Ad053324F31b --json

# Start HTTP API server (for advanced integration)
py 03_qubo_sim.py serve 8765
# Then: GET http://localhost:8765/analyze?wallet=0x...

# Pre-compute all demo wallets
py 03_qubo_sim.py precompute
```
